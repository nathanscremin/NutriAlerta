🛡️ Definição de Pronto (DoD Técnico da Equipe)

A equipe definiu critérios rigorosos de aceitação para considerar cada entrega como "Pronta", garantindo a coerência técnica do projeto:

  1. Compilação e Tipagem Forte: O frontend em Next.js e TypeScript compila completamente via npm run build sem erros no terminal.
  2. Resiliência à Redes Externas: O pipeline do coletor intercepta instabilidades de rede e falhas de verificação SSL (comum em máquinas Windows) aplicando fallback de requisições, evitando o travamento do pipeline.
  3. Integridade de Schemas: Os dados preditivos processados pela inteligência artificial possuem exata equivalência geométrica e de nomes com a base de visualização cartográfica no cliente.

🤝 Comprovação de Validação com a Empresa Parceira

📋 Ata Resumida de Alinhamento com a Secretaria Municipal de Saúde

"Em reunião virtual realizada no dia 22/04/2026 com a Coordenadora de Vigilância Alimentar e Nutricional do município de Rio Claro, alinhamos a representatividade espacial do sistema. A secretaria apontou que unidades localizadas em distritos rurais distantes (como USF Ajapi, USF Ferraz e USF Assistência) não possuíam limites geográficos bem delimitados pelas ferramentas tradicionais, o que causaria buracos no mapa preditivo."

Resolução Técnica Validada: Implementamos a técnica de **Âncoras de Voronoi Virtuais**. O sistema renderiza os marcadores de atendimento exatamente em suas geolocalizações GPS físicas reais, mas projeta as fronteiras cloroplete no mapa utilizando posições virtuais calibradas. Isso expandiu a cobertura visual da análise de risco cobrindo distritos rurais inteiros sem distorcer os marcadores reais do mapa. O parceiro validou positivamente o layout, a precisão das regiões e o destaque interativo das áreas.

🛠️ 2. Demonstração Técnica do MVP (Funcionamento Ponta a Ponta)

O NutriAlerta funciona de forma integrada conectando Coleta, Inteligência Artificial e Visualização Web de alta fidelidade:

1. Coleta e Limpeza

Coletor SISVAN (Python) faz download paginado e mescla dados no arquivo consolidado

➔

2. Modelos de IA

desnutricao_ML e obesidade_ML calculam deltas e geram projeções 2026/2027

➔

3. Rotas de API

Backend Next.js calcula Haversine geográfico e entrega métricas em /api/data

➔

4. Visualização

Frontend renderiza mapa interativo de Voronoi e ativa painéis e IA Widget

💻 A. Interface (Frontend)

  • Construído em Next.js 15, React e Tailwind CSS com Design System Premium HSL.
  • Mapa cloroplete com Leaflet exibindo células de Voronoi para as UBSs.
  • Menu lateral com filtros rápidos de idade (0-5, 6-10 e 10-19) selecionados por padrão.
  • Camadas ativáveis de escolas públicas, parques e mercados locais.
  • Widget do Chatbot assistente integrado ao painel lateral.

📡 B. Lógica e APIs (Backend)

  • API `/api/data` lê os resultados preditivos do CSV da IA e calcula distâncias Haversine.
  • API `/api/chat` processa mensagens enviando o contexto nutricional das UBSs para orientar médicos.
  • Integração total de rotas estáticas e dinâmicas geradas no build.
  • Fallback e tratamentos para carregamento de arquivos CSV.

🧠 C. Inteligência Artificial

  • Modelos Random Forest independentes para desnutrição e obesidade.
  • Engenharia de variáveis que mede distâncias reais de UBSs até serviços urbanos.
  • Leitura automatizada do ano máximo sem datas estáticas no código.
  • Projeções preditivas para 2 anos no futuro (2026 e 2027) geradas dinamicamente.
